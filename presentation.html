Generative → Agentic 04 / 17
The Core Distinction

Generative Gives Outputs. Agentic Pursues Outcomes.

✗ Generative Era
■ Prompt in, result out
■ Great for ideation and exploration
■ Mostly single-step
■ Reactive: waits for your next prompt
■ Limited memory of context
■ Weak ownership of task completion
✓ Agentic Era
◆ Goal in, system plans the execution
◆ Reasons across multiple steps
◆ Can use tools to take real action
◆ Remembers and carries context
◆ Checks results and self-corrects
◆ Collaborates with other agents and systems

Generative: “Create a landing page.” Agentic: “Understand the user, define the message, build it, optimize for conversion, generate the code, test it, and suggest improvements.”

The Core Framework

The 5 Layers of Agentic Design

A framework for almost any AI product. You stop designing a feature and start designing a behavior system.

01
Intent  the real goal, not the surface request
02
Context  what the system needs to know to act well
03
Behavior  what it should do, step by step
04
Guardrails  what it must never do without permission
05
Feedback Loop  how it learns, recovers, and improves

Five layers turn a feature into a behavior system.

Example: Placement Assistant

Intent

Prep for interviews in 30 days

Context

Resume, role, date, weak areas

Behavior

Analyze, plan, run mock Qs

Guardrails

No fake wins, no auto-apply

Feedback Loop

Track scores, adapt the plan

Designing Intelligence Means Designing Trust

▲
AUTO

Low Risk

Routine, reversible tasks with known safe patterns.

Inline status + activity log
▶
NOTIFY

Medium Risk

Actions with significant impact that require awareness.

Toast notification + impact summary
■
APPROVE

High Risk

Irreversible or high-consequence actions that block until consent.

Modal + explicit confirm

What Strong Agent Journeys Include

Patterns From the Best Designs

01 OWNERSHIP

Explicit at Every Step

Each action has a clear owner, human or agent. No ambiguity at handoff.

02 RISK TIER

Autonomy by Consequence

Auto for low risk. Notify for medium. Explicit consent for high.

03 RATIONALE

Visible Before Action

User always knows why before the agent does something significant.

04 RECOVERY

Designed Upfront

Rollback defined before the feature ships, not post-incident.
